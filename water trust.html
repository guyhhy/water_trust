คู่มือการเชื่อมต่อ GISTDA Sphere & Disaster APIs

สถาปัตยกรรมระดับองค์กรสำหรับการดึงค่าพยากรณ์น้ำท่วมและข้อมูลดาวเทียม

แอปพลิเคชันนี้ออกแบบมาตามโครงสร้าง Schema ของระบบภูมิสารสนเทศอุทกวิทยาของไทย คุณสามารถขอคีย์ใช้งานจริงจาก GISTDA เพื่อเปลี่ยนจากข้อมูลจำลอง ไปดึงข้อมูลสดผ่าน REST API ได้ โดยศึกษาจากคำแนะนำด้านล่างนี้:

1. การสมัครรับสิทธิ์ API Key

  • เข้าสู่เว็บไซต์ GISTDA Sphere Platform
  • ลงทะเบียนสมัครสมาชิกและสร้างแอพพลิเคชันในระบบนักพัฒนา เพื่อรับรหัส API_KEY
  • หรือติดต่อกองพัฒนานวัตกรรมอุตสาหกรรมอวกาศ GISTDA ทางอีเมล innotech@gistda.or.th เพื่อขอสิทธิ์ API ระบบรายงานน้ำท่วมภัยพิบัติ (Floodcheck Platform)

2. ตัวอย่างคำขอ API (Endpoints)

API ค้นหาพื้นที่น้ำท่วมย้อนหลังรายสัปดาห์: GET https://api.sphere.gistda.or.th/services/info/disaster-flood?key=YOUR_API_KEY
ข้อมูลพื้นที่น้ำท่วมซ้ำซาก (Flood Recurrence) ด้วยละติจูด/ลองจิจูด: GET https://api.sphere.gistda.or.th/services/info/disasters/flood-recurrence?key=YOUR_API_KEY&lat=14.36&lon=100.58
โครงสร้างการตอบรับข้อมูล JSON มาตรฐาน (GISTDA Response Schema): 
{
  "status": "success",
  "meta": {
    "agency": "GISTDA",
    "updated_at": "2026-06-12T14:30:00Z"
  },
  "data": {
    "stations": [
      {
        "id": "GT-CH-02",
        "station_name_th": "สถานีลุ่มแม่น้ำเจ้าพระยา (C.2 นครสวรรค์)",
        "coordinates": { "lat": 15.7011, "lng": 100.1345 },
        "measurements": {
          "water_level_msl": 18.45,
          "bank_capacity_msl": 26.20,
          "discharge_rate_cms": 1250,
          "severity_index": 0.704
        }
      }
    ]
  }
}

ตั้งค่าการเชื่อมต่อ API Key

ป้อนรหัส API Key ของ GISTDA Sphere ของคุณที่นี่ ระบบจะใช้คีย์นี้ในการพยายามเรียกข้อมูลผ่านเซิร์ฟเวอร์จริง หรือจำลองความสมบูรณ์ในการทำธุรกรรม HTTP

หมายเหตุ: การเรียก API ตรงจากเว็บเบราว์เซอร์อาจติด CORS Policy ของ GISTDA แนะนำให้ใช้รหัสนี้ผ่าน Proxy หรือการทดสอบภายในระบบ
